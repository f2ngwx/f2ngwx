  • FTP本身可以基于Linux和Windows之间跨平台文件传输，NFS是存储，但只能用于Linux与Linux之间（windows是无法检测到NFS的）
  • FTP本身默认是基于TCP协议的，也是最基本的C/S模式，Windows下比较简单，但是在Linux下会有一些SSL加密
  • Windows常见的FTP服务器： Serv-U FTP Server （这个是早期的，bug贼多，虽然他也是基于TCP的），Filezilla_server（我们现在一般用的是这个）。
  • Filezilla官网：https://filezilla-project.org/ （目前来说好像是不支持服务器端Linux的）
  • 先把他装好，然后看到client端此时是没有连接到服务器的

  • 主角是vsftp，very sucure ，他也是开源的。
  • 本身的话是 20 和 21的端口
vim /etc/services
  • 注意21是传指令，用于连接。20是传数据，用于传输。当你启动了FTP的时候，我们是启动哪个端口？我们启动的是21端口。
  • 你启动服务是要等待连接的，不是要等待传输的。通过fsp（file service protocol）传输


那么有两种模式，一种是主动模式，一种是被动模式   

  • 主动和被动对象都是针对于服务器来说的，如果是我的主 主动去找客户端连接，那么叫做主动
  • 如果是客户我等待他来找我，那我就是被动的 
  • 那么主动的时候就两个通道----21控制，20数据。主动模式的时候是我客户端要连接到服务器的21端口。请注意了，我主动去给你发，发送用户名和密码。那么客户端此时会随机地开放一个大于1024 的端口。 叫做浮动端口----然后我的数据我的主服务端是通过20向我的浮动端口发送出去的。
  • 而被动呢？passive，是我的主服务端主动去请求我FTP的21端口，也是发送用户名和数据。但是是我们服务端随机地开放了一个大于1024的端口，为你去发送数据，这个地方就不用20了。
  • 平常生产环境一定是常用被动模式，哪有服务器天天往外飞的？----他的主要目的是要跳过防火墙。

NFS
  • nfs网络文件系统。当本地硬盘够了，外界的也够了，此时扩容就需要将远程其他的硬盘挂载到本地。
  • 红帽7以nfsv4作为默认版本，使用tcp  2049端口。
    • 文件系统:   ext3 4 xfs  nfs  cifs（smb）
    • nfs经过rpc这个服务，所以你每次重启nfs服务的时候 要先重启rpc这个服务
[101]# yum -y install rpcbind nfs-utils
[101]# systemctl start rpcbind
[101]# systemctl start nfs-server

  • 权限
ro    只读
rw    读写访问
sync    资料同步写入到内存与硬盘中
async    资料会先暂存于内存中，而非直接写入硬盘
secure    nfs通过1024以下的安全tcp/ip端口发送
insecure    nfs通过1024以上端口发送
wdelay    如果多用户要写入nfs目录则归组写入
no_wdelay    如果多个用户写入nfs则立即写入
Hide    在nfs共享目录中不共享其子目录
no_hide    共享nfs目录的子目录
subtree_check    如果共享/usr/bin/之类的子目录，强制nfs检查父目录的权限
no_subtree_check    不检查
all_squash    共享文件UID和GID映射匿名用户anonymous，适合公共目录
no_all_squash    保留共享文件UID和GID
root_squash    root用户所有请求映射成anonymous用户的权限
no_squash    root用户有根目录的完全管理访问权限
注意：在发布共享目录的格式中除了共享目录是必跟参数外，其他参数都是可选的。并且共享
目录与客户端之间及客户端与客户端之间需要使用空格符号，但是客户端与参数之间是不能有
空格的
  •  NFS客户端挂载参数的优化:
NFS高并发环境下的服务端重要优化(mount -o 参数）
 
async 异步同步，此参数会提高I/O性能，但会降低数据安全（除非对性能要求很高，对数据可靠性不要求的场合。一般生产环境，不推荐使用）
 
noatime 取消更新文件系统上的inode访问时间,提升I/O性能，优化I/O目的，推荐使用。
 
nodiratime 取消更新文件系统上的directory inode访问时间，高并发环境，推荐显式应用该选项，提高系统性能

rsize/wsize 读取（rsize）/写入（wsize）的区块大小（block size），这个设置值可以影响客户端与服
务端传输数据的缓冲存储量。一般来说，如果在局域网内，并且客户端与服务端都具有足够的内存，这个
值可以设置大一点，比如说32768（bytes）,提升缓冲区块将可提升NFS文件系统的传输能力。但设置的值也不要太大，最好是实现网络能够传输的最大值为限。

net.core.wmem_default = 8388608     #内核默认读缓存
